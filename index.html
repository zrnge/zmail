Local Security Restriction Active

Your browser blocks secure client-side module execution when opening HTML files directly via the file:// protocol (double-clicking the file locally).

🚀 Have you already pushed to GitHub?
Make sure you are visiting the live HTTPS URL in your browser's address bar (e.g., https://zrnge.github.io/zmail) instead of viewing the local file path (starting with file:///).

To enable all ZMail diagnostic capabilities:

  1. Deploy statically (Recommended): Push this folder to a GitHub repository and activate GitHub Pages in Settings for secure, private HTTPS execution.
  2. Run a local server: Open a terminal in this folder and run npx serve . or python -m http.server, then open the provided http://localhost link in your browser.
